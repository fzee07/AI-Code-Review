01 — Core Concept

Webhooks — Event-Driven Architecture

In Projects 1-4, users sent requests to your API. Here, GITHUB sends requests to your API. Your server listens and reacts when events happen in the outside world.

Projects 1-4 — Request-Driven

User initiates everything

  • User sends POST request
  • API processes immediately
  • API responds to the user
  • Predictable, synchronous flow
Project 5 — Event-Driven

GitHub initiates the flow

  • Someone opens a PR on GitHub
  • GitHub POSTs to YOUR webhook URL
  • You respond in <10 seconds
  • Review runs in the background

02 — Security Pattern

HMAC Signature Verification

When someone POSTs to your webhook URL, how do you know it's GitHub and not an attacker? Answer: cryptographic signature verification using a shared secret.

1

You set a secret on GitHub

When configuring the webhook, you enter a secret string. Both you and GitHub know it.

shared secret
2

GitHub signs the payload

GitHub creates an HMAC-SHA256 hash of the request body using the secret. Sends it in X-Hub-Signature-256 header.

HMAC-SHA256
3

Your server recreates the hash

You hash the received body with the same secret. If your hash matches GitHub's hash → it's legitimate.

timingSafeEqual
4

Match → process. No match → reject.

Uses timingSafeEqual (not ===) to prevent timing attacks that leak info about which characters matched.

constant-time comparison
SAME PATTERN EVERYWHERENot just GitHub
// Every webhook provider uses this pattern:
Stripe   → Stripe-Signature header
GitHub   → X-Hub-Signature-256 header
Slack    → X-Slack-Signature header
Twilio   → X-Twilio-Signature header

// Learn it once → use it everywhere
03 — New Pattern

External API Integration

Your API doesn't just receive data and process it. It actively calls GitHub's API to fetch PR diffs and post review comments. Your server is both a consumer and producer of external APIs.

Read from GitHub

Fetch PR Diff

  • GET /repos/:owner/:repo/pulls/:number
  • Accept: application/vnd.github.v3.diff
  • Returns unified diff format
AI Analysis

Gemini Review

  • Send diff + review rules
  • Get structured comments JSON
  • Severity + category + suggestion
Write to GitHub

Post Comments

  • POST summary comment on PR
  • POST inline comments on code lines
  • Formatted with markdown + emoji
04 — Production Pattern

Fire and Forget

GitHub expects a response within 10 seconds. But AI review takes 15-30 seconds. Solution: respond immediately, run the review in the background.

BACKGROUND PROCESSINGwebhook.service.ts
// ❌ BAD: GitHub times out waiting
const review = await reviewPR(payload);  // Takes 20+ seconds
res.json(review);  // GitHub already gave up

// ✅ GOOD: Respond immediately, review in background
reviewPR(payload).catch(err => console.error(err));
res.json({ action: "review_triggered" });  // Instant response
